# Security Policy

## Supported Versions

This project is actively maintained as a personal developer portfolio and learning project.

Security updates and fixes are applied to the latest version only.

|         Version         |     Supported     |
| ----------------------- | ----------------- |
|          latest         | :white_check_mark:|
| < latest stable version |         :x:       |

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

### Contact

Send an email to:

- **Email:** koflerphillip@outlook.com  
- **Subject:** Security Vulnerability Report — Portfolio Webpage

---

### What to include

To help with analysis and resolution, please include:

- Description of the vulnerability
- Steps to reproduce
- Affected file(s) or endpoint(s)
- Potential impact (if known)
- Suggested fix (optional but appreciated)

---

## ⏱ Response Timeline

- Initial response: within 48–72 hours
- Assessment: within 5–10 working days
- Fix timeline: depends on severity and project status

You will be updated during the process if the issue is confirmed.

---

## Scope

This security policy applies to:

- Source code in this repository
- API endpoints (if applicable)
- Dependencies used in production builds

Out of scope:

- Local development environment issues
- Third-party services (e.g. GitHub, MongoDB Atlas, hosting providers)
- Misconfiguration on user systems

---

## Notes

This is a personal portfolio project, not a commercial product.  
However, responsible disclosure is still appreciated and taken seriously.

Thank you for helping improve security.
